QT中的 namespace Ui问题

一：namespace Ui问题是什么？

1：在下面代码中的两个Main_window是处在不同命名空间中分别位于Ui命名空间和全局命名空间

2.下方是Ui命名空间的Main_window，本质上是.ui文件通过uic转换成.h文件中声明的

使用 Qt Designer 编辑的 .ui 文件是 XML 格式。
构建时，uic（User Interface Compiler）将其转换为 C++ 头文件（如 ui_main_window.h）

3.在.ui文件中，我们可以看到Main_window仅仅是public继承了Ui_Main_window(uic将.ui的xml文件转换而成)继承后的Main_window也就是处于Ui命名空间中的·，在效果上仅仅为Ui::Main_window *ui;换了个名字。

4.此处也就是全局命名空间中的Main_window类的私有成员Ui::Main_window *ui;便是处于Ui命名空间的Main_window。我们需要前向声明,才可以在全局命名空间中的class类中使用Ui::Main_window *ui;

二：namespace Ui问题 为什么是这样的？如何产生的？

1。其实这不是 C++ 语法“差”，而是历史限制 + 工程权衡的结果。
